# COVID-19 Data Analysis Mini Project Report

## 1. Problem Statement & Dataset Selection (Day 21)
This project analyzes global COVID-19 tracking data to identify growth patterns, geographical infection hotspots, and outcome relationships. The project utilizes a structured pipeline comprising raw input files, SQL querying blocks, and Python visualizations to convert numbers into clear action points.

## 2. Data Cleaning & Preprocessing (Day 22)
- Implemented robust outlier handling to remove invalid records.
- Standardized tracking dates into datetime formats.
- Handled missing numeric metrics across `total_cases` and `new_cases` securely.

## 3. Exploratory Data Analysis & Statistical Insights (Day 23 & 25)
- Computed statistical distribution indicators across geographical centers.
- Extracted global trends identifying variations in infection acceleration timelines.
- Ranked international statistics to determine peak active pressure records.

## 4. SQL Analysis Summaries (Day 24)
Using targeted aggregation queries, database processing routines successfully extracted:
- Country-specific high-water marks for confirmed case volumes.
- Aggregated time-series trend baselines across global tracking windows.

## 5. Visual Narrative & Dashboard Details (Day 26 & 27)
